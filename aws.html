‚òÅÔ∏è 1. Cloud Concepts

  • Cloud Computing: On-demand delivery of IT resources over the internet with pay-as-you-go pricing.
  • 6 Advantages of Cloud:
    1. Trade capital expense (CapEx) for variable expense (OpEx).
    2. Benefit from massive economies of scale.
    3. Stop guessing capacity.
    4. Increase speed and agility.
    5. Stop spending money running and maintaining data centers.
    6. Go global in minutes.
  • Deployment Models:
    ‚Ä¢ Public Cloud: Everything built on the cloud provider (e.g., AWS, Azure, GCP).
    ‚Ä¢ Private Cloud (On-premises): Resources deployed in your own data center using virtualization.
    ‚Ä¢ Hybrid Cloud: A mix of public and private clouds connected together.
  • Service Models:
    ‚Ä¢ IaaS (Infrastructure as a Service): Basic building blocks for cloud IT (e.g., EC2, EBS). You manage the OS and applications.
    ‚Ä¢ PaaS (Platform as a Service): AWS manages the underlying infrastructure and OS; you focus on deploying your code (e.g., Elastic Beanstalk, Lambda).
    ‚Ä¢ SaaS (Software as a Service): Completed product run and managed by the service provider (e.g., Gmail, Office 365, or AWS's own Chime).

üí≤ 2. Cloud Economics and Billing

  • Pricing Models:
    ‚Ä¢ On-Demand: Pay by the hour/second with no long-term commitment.
    ‚Ä¢ Reserved Instances: 1 or 3-year term for a significant discount.
    ‚Ä¢ Savings Plans: 1 or 3-year term, offers more flexibility than RIs.
    ‚Ä¢ Spot Instances: Up to 90% off for fault-tolerant workloads that can be interrupted.
    ‚Ä¢ Dedicated Hosts: A physical server dedicated to your use, for compliance needs.
  • Billing & Cost Tools:
    ‚Ä¢ AWS Free Tier: Offers free usage for certain services up to a specific limit for 12 months, plus an "Always Free" tier.
    ‚Ä¢ AWS Organizations: Centrally manage billing and access for multiple AWS accounts. Allows for Consolidated Billing.
    ‚Ä¢ AWS Cost Explorer: Visualize, understand, and manage your AWS costs and usage over time.
    ‚Ä¢ AWS Budgets: Set custom budgets that alert you when costs exceed your thresholds.
    ‚Ä¢ AWS Pricing Calculator: Estimate the cost of your solution architecture.
    ‚Ä¢ TCO Calculator: Total Cost of Ownership. Compares the cost of running your workload on-premises vs. on AWS.
  • Support Plans:
    ‚Ä¢ Basic: Free. No tech support.
    ‚Ä¢ Developer: Business hours tech support. ~$29/mo.
    ‚Ä¢ Business: 24/7 support, faster response times. ~$100/mo. Access to Trusted Advisor full checks.
    ‚Ä¢ Enterprise: Includes a designated Technical Account Manager (TAM). ~$15,000/mo.

üåé 3. AWS Global Infrastructure

  • Region: A physical geographic location in the world with multiple Availability Zones. Regions are isolated from each other.
  • Availability Zone (AZ): One or more discrete data centers with redundant power, networking, and connectivity. They are physically separate but connected with low-latency links.
  • Edge Location: A site that CloudFront (CDN) uses to cache copies of your content closer to end-users for faster delivery.
  • High Availability: Designing systems to remain operational by using multiple AZs. If one AZ fails, the application still runs in another.
  • Fault Tolerance: A property that enables a system to continue operating properly in the event of the failure of some of its components.
  • Scalability vs. Elasticity:
    ‚Ä¢ Scalability: Ability to handle growing amounts of work (scale up/out).
    ‚Ä¢ Elasticity: Ability to automatically acquire resources as you need them and release them when you no longer do (e.g., Auto Scaling).

üîê 4. AWS Cloud Security

  • Shared Responsibility Model:
    ‚Ä¢ AWS is responsible for "Security OF the Cloud": Securing the global infrastructure, hardware, software, networking, and facilities.
    ‚Ä¢ Customer is responsible for "Security IN the Cloud": Data security, identity and access management, OS/network configuration, firewall rules.
  • IAM (Identity and Access Management): Manage access to AWS services securely. Principle of Least Privilege is key.
    ‚Ä¢ Users: End users (people, applications).
    ‚Ä¢ Groups: A collection of users with shared permissions.
    ‚Ä¢ Roles: An identity with permissions that can be assumed by a user or service. Best for temporary access.
    ‚Ä¢ Policies: JSON documents that define permissions.
  • Security Services:
    ‚Ä¢ AWS Shield: Managed DDoS (Distributed Denial of Service) protection. Standard (free) and Advanced tiers.
    ‚Ä¢ AWS WAF (Web Application Firewall): Protects web applications from common web exploits (e.g., SQL injection, cross-site scripting) at the application layer.
    ‚Ä¢ AWS KMS (Key Management Service): Create and manage encryption keys.
    ‚Ä¢ Amazon Inspector: Automated security assessment service for EC2 instances.
    ‚Ä¢ Amazon GuardDuty: Intelligent threat detection service that monitors for malicious activity.
    ‚Ä¢ Amazon Macie: A data security service that uses machine learning to discover, classify, and protect sensitive data (like PII) in S3.

üåê 5. Networking and Content Delivery

  • VPC (Virtual Private Cloud): Your own logically isolated section of the AWS Cloud where you can launch resources in a virtual network you define.
  • VPC Components:
    ‚Ä¢ Subnet: A range of IP addresses in your VPC. Can be Public (access to internet) or Private.
    ‚Ä¢ Internet Gateway: Allows communication between your VPC and the internet.
    ‚Ä¢ Security Group: A stateful firewall for EC2 instances. Acts at the instance level.
    ‚Ä¢ NACL (Network Access Control List): A stateless firewall for subnets. Acts at the subnet level.
  • Route 53: A scalable and highly available Domain Name System (DNS) web service. Can also perform health checks.
  • CloudFront: Content Delivery Network (CDN). Caches content at Edge Locations to deliver it to users with low latency.
  • Direct Connect: A dedicated, private physical network connection from your on-premises data center to AWS.

üíª 6. Compute

  • EC2 (Elastic Compute Cloud): A virtual server in the cloud. Provides secure, resizable compute capacity.
  • Elastic Beanstalk: A PaaS service for deploying and scaling web applications. You upload your code, and Beanstalk handles the deployment, capacity provisioning, load balancing, and auto-scaling.
  • Lambda: A serverless compute service. Run code without provisioning or managing servers. You pay only for the compute time you consume. It's event-driven.
  • Containers:
    ‚Ä¢ ECS (Elastic Container Service): A highly scalable, high-performance container orchestration service.
    ‚Ä¢ Fargate: A serverless compute engine for containers. Works with ECS and EKS. You don't manage the underlying EC2 instances.
    ‚Ä¢ EKS (Elastic Kubernetes Service): A managed service to run Kubernetes on AWS.

üíæ 7. Storage

  • S3 (Simple Storage Service): Infinitely scalable object storage. Data is stored in Buckets. Highly durable (11 nines).
  • S3 Storage Classes:
    ‚Ä¢ S3 Standard: For frequently accessed data.
    ‚Ä¢ S3 Intelligent-Tiering: Automatically moves data to the most cost-effective tier.
    ‚Ä¢ S3 Standard-IA / One Zone-IA: For infrequently accessed data.
    ‚Ä¢ S3 Glacier (Instant, Flexible, Deep Archive): For long-term data archiving. Very low cost.
  • EBS (Elastic Block Store): Persistent block-level storage volumes for use with EC2 instances (like a virtual hard drive). Tied to a specific AZ.
  • EFS (Elastic File System): A managed, scalable file storage service for use with EC2. Can be mounted on multiple instances across multiple AZs.
  • Storage Gateway: A hybrid cloud storage service that gives you on-premises access to virtually unlimited cloud storage.

üóÑÔ∏è 8. Databases

  • RDS (Relational Database Service): A managed service for relational databases. Supports engines like MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, and Amazon Aurora.
  • Amazon Aurora: AWS's own high-performance, fully managed relational database. Compatible with MySQL and PostgreSQL.
  • DynamoDB: A fast, flexible, and highly scalable NoSQL key-value database. Fully managed and serverless.
  • ElastiCache: An in-memory data store or cache service. Improves application performance by retrieving data from fast, managed, in-memory caches. Supports Redis and Memcached.
  • Redshift: A fast, petabyte-scale data warehouse service for analytics.

üèõÔ∏è 9. Cloud Architecture

  • Well-Architected Framework: A set of best practices for designing and operating reliable, secure, efficient, and cost-effective systems in the cloud.
    ‚Ä¢ Five Pillars: Operational Excellence, Security, Reliability, Performance Efficiency, and Cost Optimization.
  • Design Principles:
    ‚Ä¢ Design for failure: Assume components will fail and build resilient systems.
    ‚Ä¢ Decouple components: Use services like SQS (Simple Queue Service) and SNS (Simple Notification Service) to separate application tiers.
    ‚Ä¢ Implement elasticity: Use services like Auto Scaling and Elastic Load Balancing (ELB).
  • Elastic Load Balancing (ELB): Automatically distributes incoming application traffic across multiple targets, such as EC2 instances.

üìà 10. Auto Scaling and Monitoring

  • CloudWatch: A monitoring and observability service.
    ‚Ä¢ Metrics: Collects monitoring data from AWS resources.
    ‚Ä¢ Alarms: Trigger notifications based on metric thresholds.
    ‚Ä¢ Logs: Centralize logs from your systems and applications.
    ‚Ä¢ Events: Delivers a near real-time stream of system events that describe changes in AWS resources.
  • CloudTrail: Monitors and records account activity across your AWS infrastructure. Think of it as an audit trail for all API calls ("who did what, and when?").
  • Auto Scaling: Automatically adds or removes EC2 instances according to conditions you define. Helps ensure you have the correct number of instances available to handle your application's load.

üõ†Ô∏è 11. Management & Governance

  • AWS Trusted Advisor: An online tool that provides real-time guidance to help you provision your resources following AWS best practices across cost optimization, performance, security, and fault tolerance.
  • AWS CloudFormation: An Infrastructure as Code (IaC) service. Allows you to model and provision your AWS resources using templates (JSON/YAML).
  • AWS Organizations: Helps you centrally govern your environment as you grow. Manage multiple accounts, apply policies (SCPs - Service Control Policies), and simplify billing.
  • AWS Systems Manager: Gives you visibility and control of your infrastructure on AWS. You can view operational data and automate tasks across your resources.

üöö 12. Migration & Transfer

  • AWS DMS (Database Migration Service): Helps you migrate databases to AWS easily and securely. The source database remains fully operational during the migration.
  • AWS Snow Family: Services for physical data transport.
    ‚Ä¢ Snowcone: Smallest device, ~8 TB of usable storage.
    ‚Ä¢ Snowball Edge: Petabyte-scale data transport with on-board storage and compute capabilities.
    ‚Ä¢ Snowmobile: Exabyte-scale data transfer service using a 45-foot long shipping container.
