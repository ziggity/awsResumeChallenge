üêß 1. Introduction & Command Line

  • Linux: An open-source operating system kernel. A distribution (distro) like Ubuntu or Amazon Linux combines the kernel with other software to create a full OS.
  • The Shell: A command-line interface (CLI) that interprets and executes your commands. Bash is the most common shell.
  • ls: Lists directory contents. Use ls -la for a detailed, long-format list including hidden files.
  • pwd: Print Working Directory. Shows your current location in the filesystem.
  • cd [directory]: Change Directory. Use cd .. to go up one level, and cd ~ or just cd to go to your home directory.
  • man [command]: Displays the manual page for a command.
  • clear: Clears the terminal screen.

üë• 2. Users and Groups

  • Root User: The superuser with administrative privileges over the entire system. Use with caution.
  • sudo [command]: Superuser Do. Executes a command with root privileges.
  • useradd [username]: Creates a new user account.
  • passwd [username]: Sets or changes the password for a user.
  • groupadd [groupname]: Creates a new group.
  • usermod -aG [group] [user]: Adds a user to a supplementary group. The -a (append) and -G (groups) flags are crucial.
  • /etc/passwd: File containing user account information.
  • /etc/group: File containing group information.

üìÅ 3. File System & Working with Files

  • File System Hierarchy: A tree-like structure of directories. Key directories include / (root), /home, /etc (configuration), /var (variable data like logs), and /tmp (temporary files).
  • touch [filename]: Creates an empty file or updates the timestamp of an existing file.
  • mkdir [dirname]: Creates a new directory.
  • cp [source] [destination]: Copies files or directories. Use cp -r for recursive copy (for directories).
  • mv [source] [destination]: Moves or renames files or directories.
  • rm [filename]: Removes (deletes) a file. Use rm -r to remove a directory and its contents. Be careful, there is no undo!
  • cat [filename]: Concatenate and display file content.
  • less [filename]: View file content one page at a time. More powerful than cat for large files.
  • Text Editors:
    ‚Ä¢ nano: A simple, beginner-friendly command-line text editor.
    ‚Ä¢ vim / vi: A powerful but complex modal text editor. Has insert mode and command mode.

üîê 4. Managing File Permissions

  • Permission Groups: Each file has permissions for three groups: User (the owner), Group (the group owner), and Other (everyone else).
  • Permission Types:
    ‚Ä¢ Read (r): View file contents or list directory contents. (Value: 4)
    ‚Ä¢ Write (w): Modify file contents or create/delete files in a directory. (Value: 2)
    ‚Ä¢ Execute (x): Run a file as a script or enter a directory. (Value: 1)
  • chmod [permissions] [file]: Change mode. Modifies file permissions.
    ‚Ä¢ Symbolic method: chmod u+x,g-w file.txt (Add execute for user, remove write for group).
    ‚Ä¢ Octal (numeric) method: chmod 755 script.sh (rwx for user, r-x for group and other).
  • chown [user]:[group] [file]: Change owner. Modifies the ownership of a file or directory.

‚öôÔ∏è 5. Managing Processes & Services

  • Process: Any running program or command on the system.
  • ps aux: Lists all running processes on the system.
  • top / htop: Displays a real-time, interactive view of system processes and resource usage.
  • kill [PID]: Sends a signal to a process to terminate it. PID is the Process ID.
  • Service / Daemon: A program that runs in the background to perform a specific function (e.g., a web server).
  • systemctl [action] [service]: The primary tool for managing services (daemons) on modern Linux systems.
    ‚Ä¢ Actions: start, stop, restart, status, enable (start on boot), disable.

BASH 6. The Bash Shell & Scripting

  • Bash Script: A plain text file containing a series of commands. It must start with a "shebang": #!/bin/bash.
  • Variables: Store data. Assign with VAR="value" and access with $VAR.
  • Piping `|`: Sends the output of one command as the input to another. Example: ls -l | less.
  • Redirection `>` & `>>`:
    ‚Ä¢ >: Redirects output to a file, overwriting it.
    ‚Ä¢ >>: Appends output to a file.
  • grep [pattern] [file]: Searches for a pattern within a file or input stream.
  • Environment Variables: System-wide variables like $PATH, $HOME, $USER. View all with the env command.

üì¶ 7. Software Management & Logs

  • Package Manager: A tool that automates the process of installing, updating, and removing software packages.
    ‚Ä¢ apt (Debian/Ubuntu): apt update, apt install [package].
    ‚Ä¢ yum / dnf (Red Hat/CentOS/Amazon Linux): yum install [package].
  • Log Files: Records of system events, errors, and other important information. Typically located in /var/log.
  • dmesg: Displays kernel ring buffer messages, useful for diagnosing hardware issues.
  • journalctl: Queries and displays logs from the systemd journal. Use journalctl -u [service] to view logs for a specific service.
  • tail -f [logfile]: Follows a log file in real-time, showing new entries as they are added.
